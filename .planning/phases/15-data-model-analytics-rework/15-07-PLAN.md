# Plan 15-07: Data Entry Enhancements

**Requirement**: FR-13.6 (Data Entry Enhancements)
**UAC**: UAC-M3-6 (validation)
**Wave**: 3 (Can run parallel with Plan 15-06)

## Goal
Add quick entry mode for batch-adding readings, enhance reading validation with real-time feedback, and standardize delete confirmation dialogs using a shared widget.

## Current State
- Reading validation: Providers already validate `new >= previous` for cumulative meters
- Delete confirmation: Inline AlertDialog exists in all meter screens (inconsistent styling)
- No quick entry mode â€” dialog closes after each save

## Tasks

### T1: Shared delete confirmation widget + quick entry base
**Files**: `lib/widgets/dialogs/confirm_delete_dialog.dart` (NEW), `lib/widgets/dialogs/reading_form_base.dart` (NEW)
1. Create `ConfirmDeleteDialog` widget:
   - Title: localized "Delete {item}?"
   - Content: localized "This action cannot be undone."
   - Cancel (TextButton) + Delete (FilledButton, error color)
   - Static `show()` method returning `Future<bool>`
2. Create `QuickEntryMixin` (mixin for reading form dialog states):
   - Adds "Save & Next" button alongside "Save"
   - Tracks entry count, shows in dialog title: "Add Reading (3)"
   - On "Save & Next": calls save callback, clears value field, shows brief success indicator
   - On "Save": calls save callback and closes dialog

### T2: Apply quick entry and validation to all reading forms
**Files**: `lib/widgets/dialogs/electricity_reading_form_dialog.dart`, `lib/widgets/dialogs/gas_reading_form_dialog.dart`, `lib/widgets/dialogs/water_reading_form_dialog.dart`, `lib/widgets/dialogs/heating_reading_form_dialog.dart`
1. Apply `QuickEntryMixin` to all 4 reading form dialog states
2. Enhance validation in all 4 forms:
   - Accept `previousValue` parameter (nullable double) on dialog open
   - Display previous value as reference: "Vorheriger Wert: 1.234,5 kWh" / "Previous: 1,234.5 kWh"
   - Real-time validation: if entered value < previous, show inline error under field
   - Disable save buttons when validation fails
3. For edit mode: accept `nextValue` parameter too, validate between both bounds

### T3: Replace inline delete dialogs with shared widget
**Files**: `lib/screens/electricity_screen.dart`, `lib/screens/gas_screen.dart`, `lib/screens/water_screen.dart`, `lib/screens/heating_screen.dart`, `lib/screens/smart_plug_consumption_screen.dart`
1. Replace all inline `AlertDialog` delete confirmations with `ConfirmDeleteDialog.show()`
2. Pass appropriate item type label (localized)
3. Verify consistent behavior across all 5 screens
