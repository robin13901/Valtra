# Plan 15-01: Interpolation Rework & Visibility Toggle

**Requirement**: FR-13.1 (Interpolation Rework)
**UAC**: UAC-M3-11
**Wave**: 1 (Foundation — other plans depend on correct interpolation)

## Goal
Rework interpolation so values are always calculated at the 1st of each month at 00:00, add a toggle to show/hide interpolated values in reading lists, and color-code interpolated entries.

## Current State
- `InterpolationService.getMonthlyBoundaries()` already calculates 1st-of-month boundary values
- `InterpolationService.getMonthlyConsumption()` already uses boundary differences
- No UI toggle for showing/hiding interpolated values in reading lists
- Step interpolation still exists in code (FR-12.3.7 removed it from settings UI, but code remains)
- `InterpolationMethod.step` enum value still in `models.dart`

## Tasks

### T1: Clean up step interpolation remnants
**Files**: `lib/services/interpolation/interpolation_service.dart`, `lib/services/interpolation/models.dart`, `lib/providers/interpolation_settings_provider.dart`
1. Remove `InterpolationMethod.step` enum value from `models.dart`
2. Remove step-function branch from `interpolateAt()` in `interpolation_service.dart`
3. Remove `getMethodForMeterType()` and `setMethodForMeterType()` from `InterpolationSettingsProvider` (only linear remains)
4. Clean up any callers passing method parameter — default to linear everywhere
5. Update `AnalyticsProvider._aggregateMonthlyConsumption()` to stop passing method

### T2: Add interpolated values to reading list providers
**Files**: `lib/providers/electricity_provider.dart`, `lib/providers/gas_provider.dart`, `lib/providers/water_provider.dart`, `lib/providers/heating_provider.dart`
1. Add `showInterpolatedValues` boolean state to each provider (default: `false`)
2. Add `toggleInterpolatedValues()` method
3. When enabled: compute interpolated 1st-of-month boundary values using `InterpolationService.getMonthlyBoundaries()` from the provider's readings
4. Merge interpolated values into the readings list, sorted by timestamp
5. Mark interpolated entries with an `isInterpolated: true` flag
6. Use a wrapper model (e.g., `ReadingDisplayItem`) that contains either a real reading or an interpolated value

### T3: Add interpolation toggle UI to reading screens
**Files**: `lib/screens/electricity_screen.dart`, `lib/screens/gas_screen.dart`, `lib/screens/water_screen.dart`, `lib/screens/heating_screen.dart`
1. Add a toggle button in the app bar (eye icon / filter icon) to show/hide interpolated values
2. When toggled, call provider's `toggleInterpolatedValues()`
3. Render interpolated entries with a distinct style:
   - Different background color (e.g., light purple tint using Ultra Violet at 10% opacity)
   - Small "interpolated" label or icon indicator
   - Value shown but non-editable (no edit/delete menu)
4. Interpolated entries show: date (always 1st of month, 00:00), value, "Interpoliert"/"Interpolated" label

### T4: Localize new strings
**Files**: `lib/l10n/app_en.arb`, `lib/l10n/app_de.arb`
1. Add keys: `interpolatedValue`, `showInterpolatedValues`, `hideInterpolatedValues`, `interpolated`
2. EN: "Interpolated value", "Show interpolated values", "Hide interpolated values", "Interpolated"
3. DE: "Interpolierter Wert", "Interpolierte Werte anzeigen", "Interpolierte Werte ausblenden", "Interpoliert"

### T5: Tests
**Files**: `test/services/interpolation_service_test.dart`, `test/providers/*_provider_test.dart`, `test/screens/*_screen_test.dart`
1. Update interpolation service tests: remove step method tests, verify linear-only behavior
2. Add provider tests: toggle state, interpolated values merge correctly, flag correctness
3. Add widget tests: toggle button appears, interpolated items render with distinct style, non-editable
4. Verify existing analytics tests still pass (interpolation output unchanged)
