    **Part B: Rewrite HomeScreen as tabbed scaffold with GlassBottomNav**

    1. Convert HomeScreen from `StatelessWidget` to `StatefulWidget`
    2. State manages `_currentIndex` (int, default 0)
    3. Define 5 tabs with their screen widgets:
       - Index 0: Home hub view (the redesigned hub with GlassCard items)
       - Index 1: ElectricityScreen (Strom)
       - Index 2: GasScreen (Gas)
       - Index 3: WaterScreen (Wasser)
       - Index 4: AnalyticsScreen (Analyse)
    4. Scaffold structure:
       ```dart
       Scaffold(
         appBar: buildGlassAppBar(
           context: context,
           title: _getTitle(context, _currentIndex),
           actions: [
             if (_currentIndex == 0) ...[
               const HouseholdSelector(),
               IconButton(icon: Icon(Icons.settings), onPressed: _navigateToSettings),
             ],
           ],
         ),
         body: IndexedStack(
           index: _currentIndex,
           children: [
             _buildHomeHub(context),  // Index 0
             const ElectricityScreen(),  // Index 1 - but as body-only
             const GasScreen(),  // etc
             const WaterScreen(),
             const AnalyticsScreen(),
           ],
         ),
         bottomNavigationBar: GlassBottomNav(
           currentIndex: _currentIndex,
           onTap: (index) => setState(() => _currentIndex = index),
           items: [
             BottomNavigationBarItem(icon: Icon(Icons.home), label: l10n.home),
             BottomNavigationBarItem(icon: Icon(Icons.electric_bolt), label: l10n.electricity),
             BottomNavigationBarItem(icon: Icon(Icons.local_fire_department), label: l10n.gas),
             BottomNavigationBarItem(icon: Icon(Icons.water_drop), label: l10n.water),
             BottomNavigationBarItem(icon: Icon(Icons.analytics), label: l10n.analyticsHub),
           ],
         ),
         // NO floatingActionButton -- removed per FR-12.1.2
       )
       ```
    5. IMPORTANT ARCHITECTURE NOTE: The sub-screens (Electricity, Gas, Water, Analytics) already have their own Scaffold with AppBar. When embedded in IndexedStack, this creates nested Scaffolds. Two approaches:
       - **Option A (simpler)**: Keep sub-screens as push navigation from the home hub, and the bottom nav just highlights which section the user is in. The hub view (index 0) is the main view; tapping a bottom nav item pushes that screen.
       - **Option B (recommended per research)**: The home hub stays at index 0 with GlassCard navigation tiles. Bottom nav items 1-4 push to their respective screens using Navigator.push (same as current behavior). The bottom nav acts as quick shortcuts, not as a tab controller. After navigation, the bottom nav resets to index 0 (home).

       USE Option B: The bottom nav is a shortcut bar. Tapping an item pushes the screen and resets the index to 0 after navigation completes. This avoids nested Scaffold issues entirely.

    6. Redesign the home hub view (`_buildHomeHub`):
       - Keep the Valtra icon and title at top
       - Keep _buildCurrentHousehold
       - Replace _buildCategoryChip calls with GlassCard-based navigation items
       - Layout: Use a GridView or Column of GlassCard items for all 6 categories (Electricity, Smart Plugs, Gas, Water, Heating, Analytics)
       - Each GlassCard shows: icon, label, tap to push-navigate
       - NO Divider before Analytics (FR-12.1.1)
       - NO FAB (FR-12.1.2)
       - Heating and Smart Plugs accessible from hub GlassCards (not in bottom nav, per research)

    7. Add `home` l10n key if not present (DE: "Start", EN: "Home")

    8. Keep all existing _navigateTo* methods but update them to use the new structure
