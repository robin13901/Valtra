   b. In the `main()` function, change:
      ```dart

      ```
      to:
      ```dart
      WidgetsBinding widgetsBinding = WidgetsFlutterBinding.ensureInitialized();
      FlutterNativeSplash.preserve(widgetsBinding: widgetsBinding);
      ```
      This keeps the native splash visible while Dart code runs.

   c. After `await householdProvider.init();` and before `runApp(...)`, do NOT call remove() yet. The household stream data may not have arrived.

   d. After the `runApp(...)` call, add the splash removal logic:
      ```dart
      _removeSplashWhenReady(householdProvider);
      ```

   e. Add a top-level function (outside main, outside any class):
      ```dart
      /// Removes the native splash screen once household data is loaded.
      ///
      /// HouseholdProvider.init() sets isInitialized=true and starts the DB stream,
      /// but the first stream event (containing actual household list) arrives
      /// asynchronously. We listen for the first notifyListeners() call after init
      /// to ensure the UI has data before removing the splash.
      @visibleForTesting
      void removeSplashWhenReady(
        HouseholdProvider provider, {
        void Function()? removeSplash,
      }) {
        final remove = removeSplash ?? FlutterNativeSplash.remove;

        // Provider is already initialized (init() was awaited).
        // The stream listener in HouseholdProvider will fire notifyListeners()
        // when the first batch of households arrives. We wait for that.
        // However, if there are zero households, isInitialized is true and
        // the stream still fires with an empty list — so we listen once.
        void listener() {
          provider.removeListener(listener);
          WidgetsBinding.instance.addPostFrameCallback((_) {
            remove();
          });
        }
        provider.addListener(listener);
      }
      ```

      Then in main(), call:
      ```dart
      removeSplashWhenReady(householdProvider);
      ```

      Making the function public with @visibleForTesting and accepting an optional `removeSplash` callback allows unit testing without depending on the native splash plugin.

      IMPORTANT: The function waits for ONE notifyListeners() from HouseholdProvider after init(). This works because:
      - `init()` is already awaited (isInitialized = true)
      - The DB stream subscription fires with the household list (possibly empty)
      - That stream event triggers notifyListeners()
      - On that callback, we schedule splash removal on the next frame

      Do NOT check `provider.isInitialized` — it is already true from the awaited init(). The real signal is the first stream event arriving (which triggers notifyListeners).

2. Create `test/main_splash_test.dart` to test the splash removal logic:

   ```dart
   import 'package:flutter/material.dart';
   import 'package:flutter_test/flutter_test.dart';
   import 'package:shared_preferences/shared_preferences.dart';
   import 'package:valtra/main.dart';
   import 'package:valtra/providers/household_provider.dart';
   import 'package:valtra/database/daos/household_dao.dart';

   import 'helpers/test_database.dart';

   void main() {
     group('removeSplashWhenReady', () {
       late AppDatabase db;

       setUp(() {
         SharedPreferences.setMockInitialValues({});
         db = createTestDatabase();
       });

       tearDown(() async {
         await db.close();
       });

       testWidgets('removes splash after provider notifies listeners',
           (tester) async {
         final provider = HouseholdProvider(HouseholdDao(db));
         await provider.init();

         bool splashRemoved = false;

         // Wrap in a minimal app so WidgetsBinding is available
         await tester.pumpWidget(const MaterialApp(home: SizedBox()));

         removeSplashWhenReady(
           provider,
           removeSplash: () => splashRemoved = true,
         );

         // Stream event arrives, triggers notifyListeners
         await tester.runAsync(() async {
           // Give the stream time to emit its first event
           await Future.delayed(const Duration(milliseconds: 100));
         });

         // Pump to process addPostFrameCallback
         await tester.pump();

         expect(splashRemoved, isTrue);
         provider.dispose();
       });

       testWidgets('removes splash even when no households exist',
           (tester) async {
         final provider = HouseholdProvider(HouseholdDao(db));
         await provider.init();

         bool splashRemoved = false;

         await tester.pumpWidget(const MaterialApp(home: SizedBox()));

         removeSplashWhenReady(
           provider,
           removeSplash: () => splashRemoved = true,
         );

         await tester.runAsync(() async {
           await Future.delayed(const Duration(milliseconds: 100));
         });

         await tester.pump();

         expect(splashRemoved, isTrue);
         expect(provider.households, isEmpty);
         provider.dispose();
       });
     });
   }
   ```

   The tests verify:
   - Splash is removed after the provider emits its first stream event
   - Splash is removed even with zero households (empty DB)
   - The removeSplash callback injection enables testing without the native plugin
