<task type="auto">
  <name>Task 1: Create icon asset and generate platform icons</name>
  <files>
    assets/icon/icon.png
    pubspec.yaml
  </files>
  <action>
1. Create directory `assets/icon/`.

2. Generate a 1024x1024 PNG icon programmatically using Dart. The icon should represent the Valtra brand:
   - Background: Ultra Violet gradient (#5F4A8B to #7B68A5)
   - Foreground: A simple white house silhouette with a gauge/meter arc inside it
   - Use the `dart:ui` Canvas API via a standalone Dart script, or use ImageMagick if available on the system (`magick` command). If neither is practical, create a solid Ultra Violet (#5F4A8B) square with rounded corners as a placeholder — the user can replace the asset later.
   - Save to `assets/icon/icon.png`.
   - IMPORTANT: The file must be a valid PNG, at least 1024x1024 pixels. Do NOT use a tiny placeholder.

3. Add `flutter_launcher_icons` to `dev_dependencies` in pubspec.yaml:
   ```yaml
   flutter_launcher_icons: "^0.14.4"
   ```

4. Add icon configuration block at the root level of pubspec.yaml (NOT inside `flutter:` section):
   ```yaml
   flutter_launcher_icons:
     android: "launcher_icon"
     ios: true
     image_path: "assets/icon/icon.png"
     min_sdk_android: 21
   ```

5. Run `flutter pub get` to install the package.

6. Run `dart run flutter_launcher_icons` to generate all platform icon sizes.

Note: The icon asset does NOT need to be listed in the `flutter: assets:` section of pubspec.yaml — flutter_launcher_icons reads it directly from the filesystem path. Do NOT add an assets section for the icon.
  </action>
  <verify>
- `ls assets/icon/icon.png` confirms icon file exists
- `dart run flutter_launcher_icons` completes without errors
- `ls android/app/src/main/res/mipmap-*` shows generated Android icons
- `flutter pub get` succeeds
  </verify>
  <done>
A custom app icon is generated for all Android mipmap densities and iOS AppIcon sizes from the source 1024x1024 PNG.
  </done>
</task>
