   Future<void> _handleExport(BuildContext context, BackupRestoreProvider provider) async {
     await provider.exportDatabase();
     if (context.mounted) {
       final msg = provider.state == BackupRestoreState.success
           ? AppLocalizations.of(context)!.exportSuccess
           : provider.errorMessage ?? AppLocalizations.of(context)!.importFailed;
       ScaffoldMessenger.of(context).showSnackBar(SnackBar(content: Text(msg)));
       provider.resetState();
     }
   }

**App restart strategy after import (modify `lib/main.dart`):**

Add a callback mechanism that the BackupRestoreProvider (or settings screen) can invoke to restart the app state. The approach:

1. Add a `restartApp` static method or use a `GlobalKey<NavigatorState>` pattern. The simplest approach for this app:
   - After successful import, navigate to a simple "RestartScreen" that re-runs initialization
   - OR: use `Navigator.of(context).pushAndRemoveUntil()` to the HomeScreen after re-creating providers

2. Concrete implementation: Add a `VoidCallback? onDatabaseReplaced` to `BackupRestoreProvider`. In main.dart, set this callback to a function that:
   - Closes the old AppDatabase
   - Creates a new AppDatabase(openConnection())
   - Reinitializes all providers with new DAOs from the new database
   - Calls `Navigator.pushAndRemoveUntil` to fresh HomeScreen

3. Alternative (simpler): Store a `GlobalKey<_ValtraAppState>` and call a `restartWithNewDatabase()` method on the state that swaps out the database and rebuilds. Since ValtraApp is a StatefulWidget, this works cleanly.

Choose the approach that requires the least refactoring. The callback approach is recommended since it avoids global state.
