  <action>
    Modify `LiquidGlassBottomNav` in `lib/widgets/liquid_glass_widgets.dart`:

    **1. Remove the dot indicator from `_NavItem`:**
    - Delete the `AnimatedContainer` at the bottom of the `_NavItem.build()` Column that renders the dot (the one with `height: 4, width: selected ? 4 : 0, shape: BoxShape.circle`).
    - The active tab is already indicated by the `color` parameter on both the Icon and the label Text. The `selected` bool controls fontWeight (w600 vs w400). This is sufficient -- no new indicator needed.

    **2. Integrate FAB into the navigation pill:**
    Currently the FAB row renders ABOVE the pill in a Column. The redesign places the FAB as the rightmost item INSIDE the pill.

    Restructure `LiquidGlassBottomNav.build()`:
    - Remove the separate FAB row (the Column with `_FloatingActionCircle` buttons above the pill).
    - Inside the pill's `Row` (where nav items are generated), AFTER the navigation item list, conditionally add the FAB as an additional item:
      ```dart
      // Inside the pill Container's Row children:
      ...List.generate(icons.length, (i) { /* existing nav items */ }),
      // Right FAB integrated as last pill item
      if (showRight)
        GestureDetector(
          behavior: HitTestBehavior.opaque,
          onTap: onRightTap,
          key: const Key('right_fab'),
          child: Container(
            width: 48,
            height: 48,
            margin: const EdgeInsets.symmetric(horizontal: 4),
            decoration: BoxDecoration(
              shape: BoxShape.circle,
              color: theme.colorScheme.primary.withValues(alpha: 0.15),
            ),
            child: Icon(
              rightIcon,
              color: theme.colorScheme.primary,
              size: 22,
            ),
          ),
        ),
      ```

    The FAB should blend into the nav bar visually (glass/translucent style, not accent-colored solid background). Use `theme.colorScheme.primary.withValues(alpha: 0.15)` for a subtle tinted circle that matches the glass aesthetic. In dark mode, use a slightly different alpha (0.2).

    - The left FAB (`onLeftTap`) is currently unused by any screen (all screens pass `onLeftTap: null` or `leftVisibleForIndices: const {}`). Remove the left FAB logic entirely to simplify:
      - Remove `onLeftTap`, `leftVisibleForIndices`, `keepLeftPlaceholder` parameters from the constructor.
      - Clean up related variables and the `showLeft` logic.

    **3. Adjust pill height if needed:**
    The current pill height is 56.0. With the FAB inside, it should stay at 56 since the FAB circle (48px) fits within 56px height with 4px margin.

    **4. Keep the `_FloatingActionCircle` class** for potential reuse elsewhere (it's also used by `buildCircleButton`).

    **5. Ensure the pill Row uses proper layout:**
    The nav items use `Expanded` wrappers. The FAB should NOT be Expanded -- it should be a fixed-width element at the end. Use a Row with the nav items in an Expanded wrapper and the FAB outside:
    ```dart
    Row(
      children: [
        Expanded(
          child: Row(
            children: List.generate(icons.length, (i) {
              return Expanded(child: /* nav item */);
            }),
          ),
        ),
        if (showRight) /* FAB widget */,
      ],
    )
    ```
    This keeps nav items evenly distributed and the FAB at a fixed width on the right.
