# Plan 15-06: Gas Analysis Fix & Yearly Analysis Rework

**Requirement**: FR-13.4 (Gas Analysis Fix), FR-13.5 (Yearly Analysis Rework)
**UAC**: UAC-M3-14, UAC-M3-15
**Wave**: 3 (Depends on Plan 01 interpolation cleanup)

## Goal
1. Display gas consumption in m³ (as entered) instead of converting to kWh in analytics.
2. Rework yearly analysis: extrapolate to year-end for current year, show previous year comparison, monthly breakdown.

## Current State
- Gas readings stored in m³, converted to kWh via `GasConversionService` in `AnalyticsProvider`
- Yearly analysis shows monthly breakdown but NO extrapolation
- Previous year comparison partially implemented in `YearlyAnalyticsData` model
- `InterpolationService` has explicit "no extrapolation" policy

## Tasks

### T1: Gas analysis — display in m³
**Files**: `lib/providers/analytics_provider.dart`
1. In `_loadMonthlyData()`: remove gas kWh conversion for display (keep raw m³)
2. In `_loadYearlyData()`: same — no gas conversion
3. In `_loadOverview()`: same — no gas conversion for summary cards
4. Gas unit string: always "m³" in analytics
5. **Keep** `GasConversionService` for cost calculation: if cost config uses €/kWh, apply conversion only for cost math
6. Update cost calculation: `cost = consumption_m3 * gasKwhFactor * pricePerKwh` OR `cost = consumption_m3 * pricePerM3`

### T2: Yearly analysis — extrapolation + enhanced UI
**Files**: `lib/services/interpolation/interpolation_service.dart`, `lib/services/analytics/analytics_models.dart`, `lib/providers/analytics_provider.dart`, `lib/screens/yearly_analytics_screen.dart`
1. Add `extrapolateYearEnd()` to `InterpolationService`:
   - Input: list of monthly consumptions so far
   - Calculate average monthly consumption
   - Project remaining months at same rate
   - Return: `ExtrapolationResult` with `projectedTotal`, `projectedMonths` (each marked `isExtrapolated`)
2. Add `isExtrapolated` flag to `PeriodConsumption` model
3. Add `extrapolatedTotal` field (nullable double) to `YearlyAnalyticsData`
4. In `AnalyticsProvider._loadYearlyData()`: for current year only, call `extrapolateYearEnd()`
5. Update `YearlyAnalyticsScreen`:
   - Summary card: "245 m³ (hochgerechnet: ~680 m³)" / "245 m³ (projected: ~680 m³)"
   - Bar chart: actual months solid, extrapolated months lighter/dashed
   - Previous year bars overlaid or side-by-side (verify existing implementation works)

### T3: Localize new strings
**Files**: `lib/l10n/app_en.arb`, `lib/l10n/app_de.arb`

2. Add: `projectedTotal` ("Projected total: {value}" / "Hochgerechnet: {value}")
