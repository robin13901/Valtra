# Plan 15-04: Heating Meter Rework — Data Layer & Room Assignment

**Requirement**: FR-13.3 (Heating Meter Rework)
**UAC**: UAC-M3-13
**Wave**: 2a (Can run parallel with Plan 15-02)

## Goal
Update heating meter data model: replace optional location with mandatory room FK, add heating type (own/central) with optional ratio. Schema changes in `tables.dart` — migration written in Plan 15-07.

## Current State
- `HeatingMeters` table: `id`, `householdId`, `name`, `location` (nullable text)
- No `roomId` FK — heating meters are flat list per household
- Rooms table exists, used only by smart plugs
- RoomDao has all CRUD operations

## Tasks

### T1: Update table definitions (NO migration — Plan 15-07 handles migration)
**Files**: `lib/database/tables.dart`
1. Add `roomId` column (int, FK → Rooms.id) to `HeatingMeters` table
2. Remove `location` column from `HeatingMeters`
3. Add `heatingType` column: text with values `own_meter` / `central_meter` (default: `own_meter`)
4. Add `heatingRatio` column (real, nullable) — percentage for central_meter (e.g., 0.25 = 25%)
5. Add `HeatingType` enum to tables.dart
6. Run `dart run build_runner build --delete-conflicting-outputs` to regenerate

### T2: Update HeatingDao
**Files**: `lib/database/daos/heating_dao.dart`
1. Update `insertMeter()` — require `roomId`, optional `heatingType` + `heatingRatio`
2. Add `getMetersForRoom(roomId)` and `watchMetersForRoom(roomId)`
3. Update `getMetersForHousehold()` — join with rooms for grouping
4. Remove location-related queries
5. Add query for meters grouped by room

### T3: Update HeatingProvider
**Files**: `lib/providers/heating_provider.dart`
1. Update meter CRUD to use roomId instead of location
2. Add room-based grouping (same pattern as SmartPlugProvider)
3. Support heatingType and heatingRatio in meter state
4. Return `HeatingMeterWithRoom` objects for display

### T4: Update AnalyticsProvider for heating ratios
**Files**: `lib/providers/analytics_provider.dart`
1. In `_loadMonthlyData()`: for `central_meter` type, multiply consumption by `heatingRatio`
2. In `_loadYearlyData()`: same ratio application
3. In `_loadOverview()`: same ratio application
4. For `own_meter` type: use full consumption as-is (no change)
5. Apply ratio AFTER interpolation (ratio scales the final consumption, not raw readings)

### T5: Tests
**Files**: `test/database/daos/heating_dao_test.dart`, `test/providers/heating_provider_test.dart`, `test/providers/analytics_provider_test.dart`
1. DAO tests: room-based queries, heatingType/ratio storage and retrieval
2. Provider tests: room-based grouping, HeatingMeterWithRoom model
3. Analytics tests: ratio multiplication for central meters, own meters unchanged
4. Edge cases: ratio = 0, ratio = 1, null ratio (own_meter)
